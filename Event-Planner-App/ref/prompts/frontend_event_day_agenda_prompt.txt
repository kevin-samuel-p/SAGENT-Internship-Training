So let's discuss activity-wise what happens on event day:
On event day, the team members, organizers and vendors get a notification that the event has started. Guests are not notified about this since they are already formally invited. When they click on this notification, it takes them to the Event Details page (which was previously hidden from the Dashboard but accessible if you go to one of the Management pages and click Back). On the Event Details page, under the existing setup, we'll create a div with the heading "Event Day Agenda" consisting of a horizontal progress bar/timeline and the list of events. The organizer also has to create this list of events. Please make sure this is stored and reflected in the database and backend. It should also be modifiable. This agenda div should be present on the page, and the timeline/checkpoint/progress bar should only be activated on event day. The organizer should be able to mark whenever a session/segment is done, and the progress bar/checkpoint bar should proceed to the next checkpoint/segment/session. After the end of the event, if all sessions/segments were completed, the bar should be fully green. Otherwise, the bar should be filled with red to the extent that it progressed, to denote that the event was interrupted or had to be canceled/postponed/discontinued.

Also, on the event details modal per event on the dashboard page, under the Chat button, create a gray color button saying "View Agenda" and make it link to the respective Event Details page.
